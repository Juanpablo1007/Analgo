Proyecto de Análisis Bibliométrico y Computacional

Este proyecto propone un análisis bibliométrico y computacional centrado en el dominio del pensamiento computacional, utilizando bases de datos científicas de la Universidad del Quindío. Los objetivos principales incluyen unificar la información científica en una fuente común, generar estadísticas descriptivas sobre la productividad académica y analizar la frecuencia de términos específicos en el contenido de artículos.

Requerimientos del Proyecto

  1. Unificación de datos: Integrar todos los registros de productos académicos en una sola fuente, evitando duplicados.
  2. Estadísticas descriptivas: Analizar variables como el autor, año de publicación, tipo de producto, entre otras, para los artículos más citados.
  3. Frecuencia de categorías: Analizar el contenido de abstracts para identificar frecuencias de términos específicos en varias categorías (habilidades, conceptos computacionales, actitudes, etc.), unificando sinónimos.
  4. Nube de palabras clave: Crear una nube de palabras generada a partir de términos encontrados en abstracts.
  5. Representación gráfica: Desarrollar un grafo que muestre la relación entre los 10 journals más relevantes y los artículos con mayor citación, añadiendo información de país y citaciones.
  6. Despliegue del proyecto: Asegurar que la aplicación esté implementada y disponible.

2. Procesamiento de Datos

El backend utiliza Pandas y NumPy para manejar y procesar grandes volúmenes de datos científicos. Estos datos se obtienen de archivos CSV y se unifican en un solo DataFrame, eliminando duplicados y asegurando la consistencia. Algunas de las operaciones incluyen:

  • Unificación de Datos: Combina registros de distintas fuentes (IEEE, Scopus, ACM, etc.) en un formato coherente.
  • Estadísticas Descriptivas: Calcula métricas como la cantidad de publicaciones por año, los autores más citados, y los journals más populares.
  • Análisis de Abstracts: Realiza conteos de términos específicos en abstracts para identificar tendencias en el pensamiento computacional.

3. Generación de Imágenes y Gráficos

Para visualizar los datos, el backend genera gráficos utilizando Matplotlib y nubes de palabras con WordCloud. Estas imágenes se convierten a formato Base64 para ser enviadas fácilmente al frontend. Ejemplos incluyen:

  • Gráficos de Barras: Muestran la distribución de artículos por año, autores, o afiliaciones.
  • Nube de Palabras: Visualiza los términos más frecuentes en abstracts de artículos.
  • Grafos de Relaciones: Utilizando NetworkX, se crea un grafo que conecta journals con sus artículos más citados y países de los autores.

4. Hilos de Ejecución

Algunas tareas, como el análisis de datos intensivos, se ejecutan en segundo plano usando hilos de Python. Esto asegura que el servidor pueda responder rápidamente a las solicitudes del usuario sin bloquearse.

Funcionamiento del Frontend

El frontend de la aplicación está construido con HTML, CSS, JavaScript y utiliza Bootstrap para el diseño. Su propósito es proporcionar una experiencia de usuario fluida e interactiva. A continuación, se describen sus componentes principales:

1. Estructura y Diseño

El frontend utiliza Bootstrap para crear una interfaz atractiva y responsive. Los elementos clave incluyen:

  • Barra de Navegación: Permite a los usuarios cambiar entre diferentes secciones del proyecto, como los distintos requerimientos.
  • Acordeones: Se utilizan para mostrar y ocultar gráficos de manera interactiva.
  • Canvas de Chart.js: Utilizado para renderizar gráficos dinámicos directamente en el navegador.

2. Comunicación con el Backend

El frontend se comunica con el backend usando solicitudes AJAX realizadas con el método fetch de JavaScript. La estructura de esta comunicación es:

  • El usuario hace clic en un botón, lo que dispara un evento click en JavaScript.
  • Se realiza una solicitud HTTP al servidor Flask, solicitando datos o gráficos.
  • El backend procesa la solicitud y responde con un objeto JSON que contiene los datos en Base64 o la información solicitada.
  • El frontend toma estos datos y los muestra en la interfaz de usuario, actualizando elementos como imágenes o gráficos.

3. Manejo de Errores y Seguridad

Se implementan varias medidas para garantizar la seguridad y la robustez del proyecto:

  • CORS: Configurado en el backend para permitir solicitudes desde diferentes dominios, asegurando que solo las fuentes permitidas puedan interactuar con la API.
  • Captura de Errores: En el frontend, se utilizan bloques try-catch para manejar errores en solicitudes HTTP y mostrar mensajes de error al usuario.

Resumen de la Interacción entre Backend y Frontend

En este proyecto, el backend y el frontend trabajan en conjunto para proporcionar una experiencia de usuario fluida y eficiente:

  1. El usuario interactúa con la interfaz del frontend, haciendo clic en botones para solicitar datos.
  2. El frontend envía una solicitud al backend Flask usando el método fetch.
  3. El backend procesa la solicitud, realiza cálculos o genera gráficos, y envía los datos de vuelta al frontend.
  4. El frontend muestra los datos al usuario, actualizando elementos como imágenes y gráficos.
